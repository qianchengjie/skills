# 切片开发 · decisions.md 文件

本文只描述目录化完整档中的 `decisions.md`。

## 职责

`decisions.md` 保存所有 `D*` 分叉正文，不管状态是 `open` 还是 `decided`。`plan.md` 的切片 `关联项` 只保存 D 的 ID 和状态摘要。

`D*` 记录真正需要用户拍板的分叉及作为 `skipped` 关闭依据的稳定范围决定。`open` 的 `问题` 是等待确认的口径分叉；`decided` 的 `结论` 是后续执行必须遵守的稳定判断。

## ID

`D*` 使用数字路径：

```text
D1
D27
D27.1
```

标题必须以 ID 开头：

```markdown
### D27：是否移除 历史 multiShop 防御代码
```

除 `### D*：...` 分叉块标题外，`decisions.md` 不允许出现其他二级或三级标题；分叉正文内部小节从四级标题开始。

## 模板

```markdown
# 分叉记录

暂无分叉。
```

新增分叉：

```markdown
### D27：<分叉标题>

- 状态：open
- 关联：S2.1.4
- 问题：<一次只问一个具体问题>
- 推荐：<推荐答案>
- 证据：<短证据；长证据引用 A*>
```

已决定分叉：

```markdown
### D27：<分叉标题>

- 状态：decided
- 关联：S2.1.4
- 结论：<已确认口径>
- 证据：<短证据；长证据引用 A*>
```

## 状态

`D*` 只允许：

- `open`：仍阻塞关联切片或计划级判断。
- `decided`：已有结论，不再阻塞；若结论影响后续多个切片，提炼到 `plan.md` 的全局约束。

## 字段规则

- 每个 D 块必须有 `状态` 和 `关联`。
- `open` 必须有 `问题` 和 `推荐`。
- `decided` 必须有 `结论`。
- `证据` 可写短证据；若引用 `A*`，对应 A 块必须存在于 `audits.md`。
- 这里的 `关联` 指 D 正文块内的人读上下文字段，只检查字段存在，不校验其中的 S/A 是否存在。`plan.md` 的切片 `关联项` 是另一字段，必须校验 ID 存在和状态一致。
- 例外：D 被切片 `跳过依据` 引用时，`证据` 必须非占位，正文 `关联` 必须包含该切片，且该 D 必须在切片 `关联项` 中写为 `decided`；脚本只检查这条结构绑定，不判断结论或证据是否足以支持跳过。

## 维护规则

- 发现新分叉时，先写 D 块，再在相关切片 `关联项` 中加入该 D。
- 用户确认或明确上游依据形成唯一口径后，把 D 状态改为 `decided`，写入最终口径，并同步更新切片 `关联项` 状态。
- 明确取消某个已计划切片时，即使结论直接来自上游依据，也要把这条稳定范围决策记录为 decided D，供切片 `跳过依据` 引用。
- `结论` 只写“后续怎么做 / 不做、按哪个范围或语义执行、哪个口径被排除”，不写过程流水。
- 如果某条记录既不需要用户拍板、也不作为 `skipped` 的稳定范围依据，就不创建 D，也不写入 `结论`。
- 不按状态分区；状态真源是 D 块内的 `状态` 行。
- 不把长审计矩阵塞进 D；使用 `A*` 放到 `audits.md`。
